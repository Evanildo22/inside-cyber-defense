## Folder Structure

Each project follows this structure:

```
project-name/
├── README.md        ← Overview, methodology, findings, tools
└── screenshots/     ← Visual evidence: tool output, terminals, diagrams
    └── .gitkeep     ← Placeholder — replace with real screenshots
```

## Publishing a Project

When a project is ready to go live:

1. Replace the placeholder content in `README.md` with the real writeup
2. Add screenshots to `screenshots/`
3. In the HTML page, replace `<section class="coming-soon-section">` with `<section class="project-content">` and add real content
4. Push to `main` — site auto-deploys to [www.evanildoribeiro.com](https://www.evanildoribeiro.com)
