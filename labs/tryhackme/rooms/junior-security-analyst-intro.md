## Overview

This room introduces the role of a Junior Security Analyst in a Security Operations Centre (SOC). It covers the analyst's day-to-day responsibilities, key tools used in the SOC environment, and how a typical SOC is structured.

---

## Key Topics Covered

- The role and responsibilities of a Tier 1 SOC Analyst
- SOC team structure — Tier 1, Tier 2, Tier 3, and SOC Manager
- Types of SOC (in-house, managed, hybrid)
- Core SOC functions: monitoring, triage, escalation, documentation
- Introduction to SIEM tools and how analysts interact with alerts
- Understanding the difference between true positives and false positives

---

## Key Takeaways

- A Tier 1 analyst's primary function is alert triage — determining whether an alert is a true or false positive and escalating appropriately
- Clear documentation is as important as technical analysis in a SOC environment
- SIEM tools are the central visibility layer — analysts live in dashboards, not raw logs
- Understanding attacker techniques (via frameworks like MITRE ATT&CK) helps analysts make faster, more accurate triage decisions

---

## MITRE ATT&CK / SOC Skills Practised

| Skill | Description |
|---|---|
| Alert triage | Prioritising and categorising security alerts |
| Escalation workflow | Knowing when and how to escalate to Tier 2 |
| Documentation | Recording findings and analyst notes accurately |
| SOC tooling | Familiarity with SIEM interfaces and dashboards |
